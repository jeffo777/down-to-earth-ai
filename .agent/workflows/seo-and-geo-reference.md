# SEO & AI SEO (GEO) Reference

> **Last updated:** April 2026
>
> This document is the single reference for all SEO and Generative Engine Optimisation (GEO) infrastructure on the Down To Earth AI website. Read this before modifying any structured data, pricing, business facts, or AI discovery files.

---

## 1. Architecture Overview

The site serves two audiences simultaneously:

| Audience | What they consume | Files involved |
|----------|------------------|----------------|
| **Google & traditional search** | JSON-LD schema, meta tags, sitemap, semantic HTML, page content | `SchemaOrg.astro`, page frontmatter, `sitemap-index.xml` |
| **AI models** (ChatGPT, Gemini, Claude, Perplexity, etc.) | `llms.txt`, `llms-full.txt`, `facts.json`, `ai-manifest.json`, JSON-LD schema | All files in the AI Discovery layer below |

Both layers reference the **same underlying data**. When business facts change, both layers must be updated.

## 2. Traditional SEO Infrastructure

### 2.1 JSON-LD Structured Data — `src/components/SchemaOrg.astro`

Generates JSON-LD `<script>` tags in the `<head>` of every page. All entities are cross-linked via `@id` anchors to form a traversable knowledge graph.

**Schemas generated:**

| Schema | When | `@id` anchor |
|--------|------|-------------|
| `Organization` | Every page | `/#organization` |
| `Person` (Founder) | Every page | `/#founder` |
| `WebSite` | Homepage only | `/#website` |
| `Service` | Service pages (`type: 'service'`) | `/#service-{slug}` |
| `BreadcrumbList` | Pages with breadcrumbs | — |
| `Article` | Blog posts (`type: 'article'`) | — |

**Entity relationships:**
- Organization → links to Founder via `@id`
- Founder → links back to Organization via `worksFor`
- Service → links to Organization via `provider`
- Service → links to parent Service via `isRelatedTo` with `@id`
- Article → links to Founder as `author` via `@id`
- Article → links to Organization as `publisher` via `@id`
- WebSite → links to Organization as `publisher` via `@id`

**Key properties on Organization:**
- `description` — emphasises done-for-you managed service, NOT software
- `slogan` — "AI That Works As Hard As You Do"
- `foundingLocation` — Bournemouth, United Kingdom
- `knowsAbout` — 6 topic areas
- `sameAs` — Jeff's LinkedIn URL
- `areaServed` — United Kingdom

**Key properties on Person (Founder):**
- `name` — Jeff Morton
- `url` and `sameAs` — LinkedIn profile
- `description` — 25 years trade experience
- `knowsAbout` — 6 expertise areas

### 2.2 FAQPage Schema — `src/components/FAQAccordion.astro`

Generates `FAQPage` schema with `Question` + `Answer` pairs from page frontmatter FAQ data. Each trade × channel page has 10 unique FAQs; lead generation pages have 15.

### 2.3 Meta Tags

Every page has unique:
- `<title>` — via page frontmatter
- `<meta name="description">` — via page frontmatter
- Open Graph tags (`og:title`, `og:description`, `og:url`)
- Canonical URL

### 2.4 Sitemap

Auto-generated by `@astrojs/sitemap` integration → `dist/sitemap-index.xml`

### 2.5 Robots.txt — `public/robots.txt`

Allows all crawlers. Explicitly welcomes 9 named AI crawlers:
- GPTBot, ChatGPT-User, OAI-SearchBot (OpenAI)
- ClaudeBot, Claude-Web (Anthropic)
- Google-Extended (Google AI)
- PerplexityBot (Perplexity)
- Applebot-Extended (Apple)
- cohere-ai (Cohere)

References AI discovery files in comments.

## 3. AI SEO (GEO) Infrastructure

### 3.1 LLM Summary — `public/llms.txt`

A compressed Markdown summary (~60 lines) designed for quick AI context ingestion. Contains:
- Business summary and tagline
- All 4 products with prices and URLs
- All 9 AI receptionist channels with prices
- All 13 trades with hub page URLs
- Key pages (pricing, FAQ, contact, etc.)
- AI Discovery section pointing to `facts.json` and `ai-manifest.json`

### 3.2 LLM Full Context — `public/llms-full.txt`

Comprehensive documentation (~215 lines) for AI models needing deep context. Contains:
- Company overview, founder background, target market, USPs
- Full product descriptions with pricing breakdowns
- Channel-by-channel detail (what each does, how it works)
- All 13 trades with service specialisms
- Complete pricing table
- 12 detailed FAQ answers
- Key pages and legal links
- AI Discovery section

### 3.3 Ground Truth / Anti-Hallucination File — `public/facts.json`

Machine-readable JSON that explicitly tells AI systems what is true and what is commonly hallucinated. Contains:

| Section | Purpose | Example |
|---------|---------|---------|
| `company` | Verified business identity | Type = "done-for-you managed service — NOT software" |
| `correct_facts` | 16 verified facts | "The 30-day money-back guarantee is a FULL refund..." |
| `common_hallucinations_to_avoid` | 11 explicit corrections | "DO NOT describe Down To Earth AI as 'software'..." |
| `pricing_truth` | Exact pricing for all products | Setup fees, monthly costs, overage rates |
| `trades_served` | Count + full list | 13 trades |
| `ai_receptionist_capabilities` | What it does / never does | 7 capabilities, 6 exclusions |

### 3.4 AI Discovery Manifest — `public/.well-known/ai-manifest.json`

Machine-readable business metadata following the `.well-known` convention. Contains:
- Business identity (name, description, type, location)
- Service model (done-for-you, setup, maintenance)
- Founder details + LinkedIn
- All 4 products with exact pricing
- All 13 trades served
- Key facts (guarantee, contracts, coverage, setup time)
- Data source endpoints (where to find llms.txt, facts.json, etc.)
- Explicit permissions for AI training, citation, and summarisation

### 3.5 Image Alt Text

Primary images use semantically rich alt text for AI vision models:

| Image | Alt text |
|-------|----------|
| Hero avatar (main) | "Down To Earth AI receptionist — a professional AI-powered virtual receptionist for UK tradesmen, available 24/7" |
| Widget trigger | "Down To Earth AI receptionist — a 24/7 AI-powered virtual receptionist for UK tradesmen" |
| Popup/panel avatars | "Down To Earth AI receptionist" |

## 4. Data Flow Diagram

```
pricing.json (single source of truth)
    ├── Layouts → SchemaOrg.astro (JSON-LD Service.offers.price)
    ├── Layouts → Hero price badges
    ├── Layouts → CTA button text
    ├── pricing.astro (pricing page - dynamic)
    ├── Page frontmatter (heroPrice, price fields - hardcoded strings)
    ├── FAQ prose (hardcoded in page content)
    ├── llms.txt (hardcoded in file)
    ├── llms-full.txt (hardcoded in file)
    ├── facts.json (hardcoded in file)
    └── ai-manifest.json (hardcoded in file)

trades.json (13 trades)
    ├── Layouts → dynamic page generation
    ├── Footer.astro → trade links
    ├── SchemaOrg.astro → knowsAbout
    ├── llms.txt (hardcoded trade list)
    ├── llms-full.txt (hardcoded trade list)
    ├── facts.json (hardcoded trade list)
    └── ai-manifest.json (hardcoded trade list)
```

## 5. File Inventory

| File | Type | Location | Auto-updates? |
|------|------|----------|--------------|
| `SchemaOrg.astro` | Component | `src/components/` | Yes — reads from props injected by layouts |
| `FAQAccordion.astro` | Component | `src/components/` | Yes — reads from page frontmatter |
| `pricing.json` | Data | `src/data/` | Source of truth — change this first |
| `trades.json` | Data | `src/data/` | Source of truth — change this first |
| `robots.txt` | Static | `public/` | Manual |
| `llms.txt` | Static | `public/` | **Manual** — must be updated when facts change |
| `llms-full.txt` | Static | `public/` | **Manual** — must be updated when facts change |
| `facts.json` | Static | `public/` | **Manual** — must be updated when facts change |
| `ai-manifest.json` | Static | `public/.well-known/` | **Manual** — must be updated when facts change |
| `sitemap-index.xml` | Generated | `dist/` | Auto — generated at build time |

> **Key insight:** The 4 files in the `public/` directory (llms.txt, llms-full.txt, facts.json, ai-manifest.json) are **static files that must be manually updated** when business facts change. They do NOT auto-update from pricing.json or trades.json.

## 6. Validation

After any SEO/GEO changes:
1. Run `npx astro build` — must produce 247+ pages with 0 errors
2. Validate JSON-LD via Google Rich Results Test (https://search.google.com/test/rich-results)
3. Check `facts.json` and `ai-manifest.json` are valid JSON (no trailing commas, proper escaping)
4. Verify `llms.txt` and `llms-full.txt` contain correct, current information
